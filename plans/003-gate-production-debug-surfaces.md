## Why this matters

The root route imports and renders React Query and TanStack Router devtools unconditionally. The app also ships a directly navigable Stronghold playground that can insert, read, remove, and display live vault values. These are useful during development, but they should not be present in production desktop builds that handle financial data and secrets.

## Steps

### Step 1: Gate root devtools behind development mode

Update `src/routes/__root.tsx` so devtools are only loaded and rendered in development.

Preferred shape:

- Avoid static production imports when possible. Use lazy/dynamic imports or a small `Devtools` component that renders only when `import.meta.env.DEV`.
- Preserve `Toaster` in all builds.

Minimum acceptable shape if dynamic imports are too invasive:

```tsx
{import.meta.env.DEV ? (
  <>
    <ReactQueryDevtools initialIsOpen={false} buttonPosition="bottom-left" position="bottom" />
    <TanStackRouterDevtools position="bottom-left" />
  </>
) : null}
```

If using the minimum shape, confirm whether production bundle still includes devtools packages. Prefer excluding them entirely.

**Verify**: `pnpm tsc:check` → exit 0.

### Step 2: Remove or dev-gate the Stronghold playground route

Choose the smallest production-safe option:

1. Delete `src/routes/settings/playgrounds/index.tsx` entirely if the maintainer does not need it.
2. Rename/move it to a dev-only route pattern if TanStack Router supports exclusion in this project.
3. Keep the file route but make the route component return a 404-like dev-only message in production.

Preferred result: the production route tree should not include `/settings/playgrounds/`. If the route must remain for technical reasons, it must not call `Stronghold.init()` or expose read/write controls in production.

Do not leave the current form reachable in production.

**Verify**: `pnpm generate-routes` → exit 0; inspect `src/routeTree.gen.ts` and confirm `/settings/playgrounds/` is absent if the route was removed.

### Step 3: Confirm direct navigation behavior

Run the app in dev and verify the intended dev behavior still works:

- In development, devtools appear if they are intentionally kept for dev.
- If the playground was deleted, `/settings/playgrounds/` is gone in dev too.
- If the playground was dev-gated, `/settings/playgrounds/` works only in dev.

For production, build the app and inspect the generated route tree/bundle output enough to confirm the playground UI is not available.

**Verify**: `pnpm build` → exit 0.

### Step 4: Search for remaining production debug leaks

Run searches for the debug surfaces:

- `rg "ReactQueryDevtools|TanStackRouterDevtools" src`
- `rg "Stronghold Playground|settings/playgrounds|JsonDisplay data=\\{stronghold\\}" src`

Expected results:

- Devtools references are either inside a dev-only dynamic import/component or absent from production code paths.
- Stronghold playground references are absent, or guarded by an explicit dev-only condition with no production vault access.

**Verify**: both `rg` commands show only expected dev-gated references.

## STOP conditions

Stop and report back if:

- The route tree or TanStack Router plugin cannot support excluding the playground without broader route architecture changes.
- Production gating requires changing navigation, settings, or Stronghold architecture outside this plan's scope.
- `pnpm build` fails because of unrelated pre-existing errors from Plan 001 not being complete.

## Maintenance notes

If developers still need secret-storage experiments, consider moving them to an untracked local recipe or documented dev-only route pattern. Reviewers should check the production bundle impact, not just JSX conditionals, because static imports can still ship debug dependencies even when components are not rendered.
